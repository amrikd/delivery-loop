# The 5 Levels of Agentic Adoption

Teams don't use AI the same way. Plotting where your team is today — and where you're going — is half the battle.

This workshop pushes teams to **L4 (Spec-Driven Team)**. Most start somewhere between L2 and L3.

---

## L1 · Prompt Curious

**The shape:** Individuals using ChatGPT or Claude.ai in a browser tab. One-off asks. Copy-paste back to the IDE.

**Signals:**
- AI is personal, not team
- No IDE integration
- Context dies with each chat
- No standards, no playbooks

**Productivity:** +10–20% on individual tasks. Zero coordination benefit.

---

## L2 · IDE Assisted

**The shape:** Copilot inline autocomplete. Claude Code or Cursor for conversational work. Still one developer, one session.

**Signals:**
- AI lives where the work lives
- Individuals have personal workflows
- No team-standard prompts or agents
- Tests, docs, reviews may or may not use AI

**Productivity:** +30–50% on individual coding tasks. Still artisan-scale.

---

## L3 · Lifecycle Assisted

**The shape:** AI touches the whole SDLC. Architecture conversations with Claude. Test generation. Doc updates. AI-assisted code review.

**Signals:**
- Workflows are emerging but informal
- Role-siloed — designers use AI separately from devs
- Team norms forming ("we use Copilot for tests")
- Integration between roles is still manual

**Productivity:** ~2× on individual work. Handoffs between roles are still the bottleneck.

---

## L4 · Spec-Driven Team **← workshop target**

**The shape:** Specs are the input. Agents compile them into coordinated outputs across roles. The designer's spec feeds the dev's agents. The dev's tests validate the designer's interactions. Handoffs are contracts, not meetings.

**Signals:**
- Specs are the source of truth (not code, not Figma in isolation)
- Agents enforce discipline (territory, quality gates, a11y checks)
- Cross-role outputs are coordinated, not artisanal
- First-time-right is common because ambiguity is caught at spec time

**Productivity:** ~5–10× on full feature delivery. Quality climbs at the same time — this is the 2x-multiplier effect.

**What makes this level different:** the bottleneck moves from code-writing to **spec-writing**. The team's performance is bounded by how well it can articulate intent, not how fast it can type.

---

## L5 · Autonomous Loops

**The shape:** Agents call agents. Self-review, auto-testing, auto-refinement. Humans set direction; agents execute and come back with PRs + test results + docs. Audit trails and guardrails are mature.

**Signals:**
- Multiple agent roles in a single pipeline (product → design → dev → test → deploy)
- Human intervention at decision points, not execution points
- Production-grade observability on agent runs
- Teams ship 10× the surface area they used to

**Productivity:** 10–20×. Organizational change (roles, process, governance) matters as much as tooling.

---

## Where is your team today?

Honest answer — most teams are at **L2 or L3**. That's fine. Today's workshop takes a representative pair from that starting point to L4 by the end of the session.

You're not learning a tool. You're learning the **discipline** that makes L4 possible:

- Write specs that agents can execute (not docs for humans to read later)
- Split specs by role so ownership is clear
- Enforce cross-spec consistency so handoffs don't drift
- Use agents as role-specific collaborators, not one generic helper

---

## How to move your team up a level

If you're at **L1 → L2:** get Copilot or Claude Code in front of everyone. Standardize the tool.

If you're at **L2 → L3:** start using AI for tests, reviews, and docs — not just code. Generate, don't just write.

If you're at **L3 → L4:** introduce specs as deliverables. Make the spec pass a gate before anyone builds. This is where today's workshop puts you.

If you're at **L4 → L5:** wire your agents together. Add observability. Invest in governance. Reshape roles.

The ladder is not linear for everyone — teams can skip levels with the right tools and leadership — but each rung compounds on the one below it.
